2. MDP 框架設計 (MDP Framework Design)
--------------------------------------------------------------------
本系統的馬可夫決策過程 (MDP) 參數設計如下：

[State Space (S)] - **連續狀態空間**
代理人直接接收感測器輸出的連續數值向量，輸入維度為 3：
- 土壤濕度 (Soil Moisture) - 連續數值狀態
- 環境溫度 (Temperature) - 連續數值狀態
- 光照強度 (Sunlight) - 連續數值狀態
不再需要像 HW1 將狀態人為劃分為 27 個 Bin，避免了資訊損失與邊界模糊的問題。

[Action Space (A)] 
代理人可執行的 7 種離散動作：
- a0: 什麼都不做 (Do nothing) 
- a1: 開啟水幫浦 (Turn ON water pump) 
- a2: 關閉水幫浦 (Turn OFF water pump)
- a3: 開啟加熱器 (Turn ON heater) 
- a4: 關閉加熱器 (Turn OFF heater)
- a5: 打開遮陽簾 (Open shade curtains)
- a6: 關閉遮陽簾 (Close shade curtains)

[Reward Function (R)] 
維持與 HW1 相同的獎勵函數設計，以確保對比的公平性：
- 處於最佳範圍 (Optimal Range)：濕度 +10，溫度 +10，光照 +5 
- 極端環境懲罰：濕度過乾 -20，溫度極端 -15 
- 資源成本懲罰：開啟水幫浦 -2，開啟加熱器 -5

[Transition Model (T)]
- 環境自然流失：每個時間步長有 20% 的機率自然流失水分，模擬天氣的不可預測性與環境變動。

3. DQN 網路結構與核心機制 (DQN Architecture & Core Mechanisms)
--------------------------------------------------------------------
本專案使用 PyTorch 實作 DQN 的核心結構：

1. **Q 網路架構 (MLP)**:
   - 輸入層：3 (對應 3 個連續環境狀態變數)
   - 隱藏層：兩層 Linear 層 (每層 128 個神經元，搭配 ReLU 激活函數)
   - 輸出層：7 (對應 7 個動作的預估 Q 值)

2. **經驗回放區 (Experience Replay Buffer)**:
   - 建立容量 10,000 的 Replay Buffer。
   - 每次 Agent 與環境互動產生的 `(state, action, reward, next_state, done)` 均會存入記憶庫。
   - 訓練時隨機抽樣 `batch_size=64` 的 Mini-batch 來更新網路，有效打散資料的時間相關性，提升訓練的穩定度。

3. **目標網路 (Target Network)**:
   - 設計與預估 Q 網路結構相同的 Target Network。
   - 每 10 個 episodes 定期更新 Target Network 的參數，避免預估目標（Q-Target）隨權重更新劇烈移動，從而穩定梯度下降的收斂過程。

5. 實驗結果與兩者成效對比分析 (Performance Analysis & Comparison)
--------------------------------------------------------------------
將 DQN 與 HW1 的 Q-Learning Baseline 進行 1000 個 Episodes 的公平對比，核心發現與效能分析如下：

1. **訓練過程表現 (Training Progress)**:
   - **Q-Learning**: 最後 200 回合平均獎勵為 431.57 分。
   - **DQN**: 最後 200 回合平均獎勵為 449.66 分。
   - 從結果來看，DQN 相較於 Q-Learning 提升 18.09 分，約為 +4.19%。這代表 DQN 的最終訓練表現略優於 Q-Learning，但兩者差距並非壓倒性，而是穩定的小幅提升。

2. **收斂與學習趨勢 (Convergence Trend)**:
   - 在訓練初期，DQN 需要累積 Replay Buffer，並透過 Mini-batch 方式訓練神經網路，因此前期表現不一定會立即超越 Q-Learning。
   - 從輸出結果來看，DQN 在第 600、800、1000 回合的平均獎勵皆高於 Q-Learning，顯示其在中後期逐漸展現連續狀態泛化的優勢。
   - 因此，本次實驗較合理的結論是：DQN 在中後期逐漸超越 Q-Learning，並取得較佳的最終平均獎勵。

3. **最終平均獎勵值 (Final Average Reward)**:
   - Q-Learning 最後 200 回合平均獎勵：431.57 分。
   - DQN 最後 200 回合平均獎勵：449.66 分。
   - DQN 比 Q-Learning 高出 18.09 分，約提升 4.19%。
   - 這說明 DQN 在本次智慧溫室控制任務中，能透過神經網路學習連續狀態與動作價值之間的關係，減少離散化狀態造成的資訊損失，進而取得略佳的控制策略。

4. **訓練穩定度 (Training Stability)**:
   - Q-Learning 使用離散化 Q-Table，優點是更新規則簡單且容易收斂，但會受到狀態分箱限制，可能無法完整保留連續感測資料中的細微差異。
   - DQN 則使用神經網路近似 Q 值，能直接接收土壤濕度、溫度與光照的連續數值，因此具有較佳的狀態泛化能力。
   - 不過，DQN 的訓練也受到 Replay Buffer、Target Network、學習率與隨機初始狀態影響，因此結果仍可能出現波動。整體而言，本次實驗顯示 DQN 表現略優，但仍有進一步調參與改良空間。

6. 模型測試階段實證分析 (Test Phase Quantitative Analysis)
--------------------------------------------------------------------
在完全關閉探索（Epsilon = 0）的條件下，DQN 代理人進行 10 回合測試，數據表現如下：

- 測試回合得分：
  - Test 1: 888.00
  - Test 2: 925.00
  - Test 3: 861.00
  - Test 4: 1075.00
  - Test 5: 974.00
  - Test 6: 964.00
  - Test 7: 824.00
  - Test 8: -258.00
  - Test 9: -204.00
  - Test 10: 824.00

- 測試平均分：687.30 分
- 測試最高分：1075.00 分
- 測試最低分：-258.00 分
- 正分回合：8/10 回合
- 負分回合：2/10 回合

### 測試結果分析

1. **DQN 具有較高的得分上限**  
   DQN 在測試階段最高分達到 1075.00 分，且多數測試回合皆能取得 800 分以上的高分，表示模型在大部分環境狀態下能學到有效的控制策略。這代表 DQN 能利用連續狀態資訊，對水幫浦、加熱器與遮陽簾等動作做出較細緻的判斷。

2. **整體測試表現良好，但仍非完全穩定**  
   10 次測試中有 8 次取得正分，測試平均達到 687.30 分，代表 DQN 的整體策略具有實用性。不過，第 8 回合與第 9 回合仍出現負分，分別為 -258.00 與 -204.00，顯示模型在某些極端初始狀態或隨機環境變動下仍可能失敗。

3. **連續狀態輸入是 DQN 的主要優勢**  
   相較於 Q-Learning 需要將土壤濕度、溫度與光照切分成有限的離散狀態，DQN 可以直接使用連續數值作為輸入，因此能保留更多環境資訊。這使得 Agent 能更細緻地判斷目前環境與最佳區間之間的差距，並選擇較合適的控制動作。

4. **未來改進方向**  
   儘管 DQN 在本次實驗中已展現比 Q-Learning 略佳的訓練表現與更高的測試得分上限，但仍存在少數測試失敗回合。未來可嘗試導入 Double DQN，以降低 Q 值過度估計問題；或使用 Prioritized Experience Replay，讓模型更常學習高誤差與極端狀態樣本，進一步提升穩定度。

結論
--------------------------------------------------------------------
本專案成功將 HW1 的 Tabular Q-Learning 升級為 Deep Q-Network（DQN），並將狀態表示方式由離散化 Q-Table 改為連續狀態向量輸入。透過 PyTorch 建立的 MLP、Experience Replay Buffer 與 Target Network，Agent 能直接學習土壤濕度、溫度與光照三個連續環境變數和動作價值之間的關係。

根據本次輸出結果，Q-Learning 最後 200 回合平均獎勵為 431.57 分，DQN 最後 200 回合平均獎勵為 449.66 分，DQN 約提升 18.09 分，提升幅度約為 4.19%。因此，本次實驗可以證明 DQN 在智慧溫室連續控制任務中具有較好的泛化能力與略佳的最終表現，但不應誇大為壓倒性勝利。

在測試階段，DQN 於 10 回合中取得 687.30 分的平均成績，最高分達到 1075.00 分，且有 8 回合取得正分，顯示其策略在大多數情況下能有效維持環境條件並累積高獎勵。然而，仍有 2 回合出現負分，代表在極端狀態或隨機環境變化下，模型仍可能控制失敗。

綜合而言，DQN 的主要價值在於能直接處理連續感測資料，避免 Q-Learning 人為離散化造成的資訊損失，並在測試階段展現較高的得分上限。未來若要進一步提升穩定性與抗極端狀態能力，可嘗試 Double DQN、Dueling DQN 或 Prioritized Experience Replay 等改良方法。
