1. 專案簡介 (Project Description)
--------------------------------------------------------------------
本專案為一個基於 Q-Learning 演算法的智慧溫室管理系統代理人 (Intelligent Agent)。該溫室種植高價值藥用植物，環境中具備土壤濕度、溫度與光照感測器，以及水幫浦、加熱器與遮陽簾等致動器。代理人的目標是在高度隨機的氣候環境中，透過強化學習找到最佳策略 (Optimal Policy)，以平衡植物健康與水電資源消耗。

--------------------------------------------------------------------
2. MDP 框架設計 (MDP Framework Design)
--------------------------------------------------------------------
本系統的馬可夫決策過程 (MDP) 參數設計如下：

[State Space (S)] 
為了利於 Tabular Q-Learning 實作，本程式直接以離散等級模擬環境狀態，將土壤濕度、溫度與光照各自表示為 3 個等級：
- 土壤濕度 (Soil Moisture): {0: Dry, 1: Optimal, 2: Wet}
- 溫度 (Temperature): {0: Low, 1: Optimal, 2: High}
- 光照 (Sunlight): {0: Low, 1: Medium, 2: High}
因此總狀態數為 3 × 3 × 3 = 27 種狀態。這種設計雖然簡化了 Q-Learning 的實作，但也會造成資訊損失，因為 Agent 無法觀察到更細緻的連續環境變化。

[Action Space (A)] 
代理人可執行的 7 種離散動作：
- a0: 什麼都不做 (Do nothing)
- a1: 開啟水幫浦，使土壤濕度上升一個等級
- a2: 關閉水幫浦，在本簡化模型中視為停止補水，不額外改變狀態
- a3: 開啟加熱器，使溫度上升一個等級
- a4: 關閉加熱器，在本簡化模型中視為停止加熱，不額外改變狀態
- a5: 打開遮陽簾 / 增加光照，使光照上升一個等級
- a6: 關閉遮陽簾 / 降低光照，使光照下降一個等級
此設計將實際致動器行為簡化為狀態等級的變化，以便聚焦於 Q-Learning 策略學習。

[Reward Function (R)] 
兼顧植物健康與成本的標量獎勵函數：
- 處於最佳範圍 (Optimal Range)：濕度 +10，溫度 +10，光照 +5 
- 極端環境懲罰：濕度過乾 -20，溫度極端 -15 
- 資源成本懲罰：開啟水幫浦 -2，開啟加熱器 -5

[Transition Model (T)]
模擬真實溫室的物理變化與隨機性：
- 執行動作後，對應的環境數值會上升或下降。
- 環境自然損耗：每個時間步長有 20% 的機率自然流失水分，模擬天氣的不可預測性。

4. 實驗結果與效能分析 (Experiment & Performance Analysis)
--------------------------------------------------------------------
為驗證 Agent 的效能與行為邏輯，本程式設計 6 組 Q-Learning 對照實驗，分別測試 Reward 權重、折扣因子、環境隨機性、探索衰減速度與學習率對策略表現的影響。根據本次 Notebook 輸出結果，核心發現如下：
1. 基準模型 (Baseline)：Gamma=0.9，Learning Rate=0.1。Agent 在訓練後期最後 200 回合平均達到 419.48 分，代表 Q-Learning 能在此簡化環境中學到基本有效的控制策略。
2. 獎勵塑形 (Reward Shaping)：將水費與電費懲罰大幅提高後，最後 200 回合平均下降至 256.12 分。測試最高分為 895.00 分，顯示過高的成本懲罰會壓低總分上限，也會改變 Agent 的策略偏好。
3. 折扣因子測試 (Gamma=0.1)：低折扣因子使 Agent 更重視短期回報。測試時最低單局跌至 -1750.00 分，顯示若 Agent 不重視長期環境變化，容易在部分狀態下出現嚴重失控。
4. 高蒸發率測試 (80% 掉水)：雖然環境更惡劣，但 Agent 會被迫學習更頻繁的補水策略。此組最後 200 回合平均達到 553.64 分，測試平均為 579.20 分，顯示在某些條件下，環境壓力反而可能促使 Agent 學到更積極的維持策略。
5. 探索衰減測試 (Epsilon Decay=0.8)：此組測試平均最高，達到 761.00 分，但仍在第 7 回合出現 -480.00 分。這代表快速降低探索率在本次隨機結果中剛好找到不錯的局部策略，但仍可能因探索不足而無法處理少見狀態。
6. 學習率測試 (Alpha=1.0)：最後 200 回合平均為 407.38 分，測試平均為 559.80 分。雖然測試表現不差，但過高學習率會使 Q 值更新過度依賴單次經驗，理論上較容易受到隨機雜訊影響，導致策略穩定性下降。
結論：Q-Learning 在智慧溫室控制任務中的表現，不只取決於演算法本身，也高度依賴 Reward 權重、折扣因子、探索策略、學習率與環境隨機性的平衡。

4-2. 模型測試階段數據與瓶頸分析 (Test Phase & Limitations)
--------------------------------------------------------------------
在 Baseline 基準模型關閉探索（Epsilon = 0）的情況下，連續執行 10 回合獨立測試，數據呈現如下：
- 完美通關回合：得分落於 782.00 ~ 1026.00 分之間 (共 5 回合)
- 失去控制回合：得分落於 -192.00 ~ -694.00 分之間 (共 5 回合)
- 測試平均得分：210.40 分

【技術性瓶頸分析】：
測試結果呈現極度明顯的「兩極化斷層」（不是拿近千分，就是倒扣幾百分）。這是傳統 Tabular Q-Learning 的天生技術瓶頸：
1. 狀態稀疏性（State Sparsity）：環境在 reset() 時會隨機初始化，若隨機到訓練時少走的「陌生區塊」，因 Q-Table 缺乏「泛化能力 (Generalization)」，Agent 會陷入盲目決策，導致一半的回合徹底失控。
2. 離散化資訊損失：連續數值粗暴切分成 27 個格子，導致 Agent 在極端狀態邊緣無法細緻微調挽救，落入惡性循環即一路扣分。這項缺陷強烈支持了 HW2 引入深度學習（DQN）以利用神經網路連續泛化特性的絕對必要性。
